# thesis-seating

修論発表の座席割り当て最適化ツール。学生をグリッド状（行=時間帯 x 列=会場）に配置し、教員の会場かぶりを回避しつつ移動回数を最小化する。

## 実行方法

```bash
uv run python solve.py
```

## 入力ファイル

プロジェクト直下に `student.tsv` を配置する。

### フォーマット

- TSV（タブ区切り）、ヘッダなし
- 各行: `学生ID <TAB> 主査 <TAB> 副査1 <TAB> 副査2`

### 例

```
student01	faculty17	faculty01	faculty11
student02	faculty14	faculty17	faculty03
student03	faculty14	faculty07	faculty01
```

### 制約

- 各学生に紐づく教員は必ず3名（主査1名 + 副査2名）
- 学生ID・教員IDは任意の文字列（ソート可能な命名を推奨）
- 会場数は3固定（A, B, C）

## 出力

- `result.tsv` - 学生別の配置結果（学生, 主査, 副査1, 副査2, 第N回, 会場）
- `result.xlsx` - Excel形式の配置結果（4シート: 学生別配置, タイムテーブル, タイムテーブル詳細, 教員スケジュール）
- 標準出力にグリッド表・学生一覧・サマリ・検証結果を表示

## 最適化の目的

1. **フル行の最大化**（第1優先）: 3会場すべてに学生を配置できる回数を最大化
2. **教員の会場移動回数の最小化**（第2優先）: 教員が会場間を移動する合計回数を最小化
3. **移動回数の均等化**（第3優先）: 同じ合計移動回数の中で、特定教員に移動が偏らない配置を選択
